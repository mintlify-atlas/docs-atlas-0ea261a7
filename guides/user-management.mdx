This guide covers user account management, including registration, authentication with JWT, admin user operations, managing addresses, and handling user favorites.

## User registration

Register new users with email and password. Users are assigned the "client" role by default.

<CodeGroup>
```bash cURL
curl -X POST https://api.grancamerino.com/api/register \
  -H "Content-Type: application/json" \
  -d '{
    "name": "John Doe",
    "email": "john@example.com",
    "password": "SecurePass123",
    "role": "client"
  }'
```

```javascript JavaScript
const response = await fetch('https://api.grancamerino.com/api/register', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({
    name: 'John Doe',
    email: 'john@example.com',
    password: 'SecurePass123',
    role: 'client'
  })
});

const data = await response.json();
console.log(data.token); // Store this token
```
</CodeGroup>

Successful response:
```json
{
  "message": "User registered successfully",
  "user": {
    "id": "user-123e4567-e89b-12d3-a456-426614174000",
    "name": "John Doe",
    "email": "john@example.com",
    "role": "client",
    "created_at": "2024-03-01T10:00:00.000000Z"
  },
  "token": "eyJ0eXAiOiJKV1QiLCJhbGciOiJIUzI1NiJ9...",
  "token_type": "bearer",
  "expires_in": 3600
}
```

<Note>
  The `role` field is optional and defaults to `"client"`. Only specify `"admin"` when creating admin accounts through admin endpoints.
</Note>

## User authentication

The API uses JWT (JSON Web Tokens) for authentication. Include the token in the `Authorization` header for protected endpoints.

### Login

<CodeGroup>
```bash cURL
curl -X POST https://api.grancamerino.com/api/login \
  -H "Content-Type: application/json" \
  -d '{
    "email": "john@example.com",
    "password": "SecurePass123"
  }'
```

```javascript JavaScript
const response = await fetch('https://api.grancamerino.com/api/login', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({
    email: 'john@example.com',
    password: 'SecurePass123'
  })
});

const { token } = await response.json();
localStorage.setItem('auth_token', token);
```
</CodeGroup>

Response:
```json
{
  "message": "Login successful",
  "user": {
    "id": "user-123",
    "name": "John Doe",
    "email": "john@example.com",
    "role": "client"
  },
  "token": "eyJ0eXAiOiJKV1QiLCJhbGciOiJIUzI1NiJ9...",
  "token_type": "bearer",
  "expires_in": 3600
}
```

### Get current user

```bash cURL
curl https://api.grancamerino.com/api/me \
  -H "Authorization: Bearer YOUR_TOKEN"
```

Response:
```json
{
  "user": {
    "id": "user-123",
    "name": "John Doe",
    "email": "john@example.com",
    "role": "client",
    "created_at": "2024-03-01T10:00:00.000000Z"
  }
}
```

### Refresh token

```bash cURL
curl -X POST https://api.grancamerino.com/api/refresh \
  -H "Authorization: Bearer YOUR_TOKEN"
```

Response:
```json
{
  "message": "Token refreshed",
  "token": "eyJ0eXAiOiJKV1QiLCJhbGciOiJIUzI1NiJ9...",
  "token_type": "bearer",
  "expires_in": 3600
}
```

### Logout

```bash cURL
curl -X POST https://api.grancamerino.com/api/logout \
  -H "Authorization: Bearer YOUR_TOKEN"
```

Response:
```json
{
  "message": "Successfully logged out"
}
```

<Note>
  Logout invalidates the current token. The user must log in again to get a new token.
</Note>

## Admin user management

Admin users can manage all user accounts, including creating, updating, and deleting users.

### List all users

```bash cURL
curl https://api.grancamerino.com/api/users \
  -H "Authorization: Bearer YOUR_ADMIN_TOKEN"
```

Supports filtering and search:

```bash cURL
# Filter by admin status
curl "https://api.grancamerino.com/api/users?is_admin=true" \
  -H "Authorization: Bearer YOUR_ADMIN_TOKEN"

# Search by name or email
curl "https://api.grancamerino.com/api/users?search=john" \
  -H "Authorization: Bearer YOUR_ADMIN_TOKEN"

# Sort and paginate
curl "https://api.grancamerino.com/api/users?sort_by=created_at&sort_order=desc&per_page=50" \
  -H "Authorization: Bearer YOUR_ADMIN_TOKEN"
```

### Create a user (Admin)

```bash cURL
curl -X POST https://api.grancamerino.com/api/users \
  -H "Authorization: Bearer YOUR_ADMIN_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "Jane Smith",
    "email": "jane@example.com",
    "password": "SecurePass456",
    "password_confirmation": "SecurePass456",
    "is_admin": false
  }'
```

Response:
```json
{
  "status": "success",
  "message": "User created successfully",
  "data": {
    "id": "user-456",
    "name": "Jane Smith",
    "email": "jane@example.com",
    "is_admin": false,
    "created_at": "2024-03-01T11:00:00.000000Z"
  }
}
```

### Update a user

```bash cURL
curl -X PUT https://api.grancamerino.com/api/users/{user_id} \
  -H "Authorization: Bearer YOUR_ADMIN_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "Jane Doe",
    "email": "jane.doe@example.com"
  }'
```

All fields are optional. To update password:

```bash cURL
curl -X PUT https://api.grancamerino.com/api/users/{user_id} \
  -H "Authorization: Bearer YOUR_ADMIN_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "password": "NewSecurePass789",
    "password_confirmation": "NewSecurePass789"
  }'
```

### Delete a user

```bash cURL
curl -X DELETE https://api.grancamerino.com/api/users/{user_id} \
  -H "Authorization: Bearer YOUR_ADMIN_TOKEN"
```

Response:
```json
{
  "status": "success",
  "message": "User deleted successfully"
}
```

<Warning>
  Admins cannot delete their own account. The API will return a 403 error if you try.
</Warning>

### Toggle admin status

```bash cURL
curl -X PATCH https://api.grancamerino.com/api/users/{user_id}/toggle-admin \
  -H "Authorization: Bearer YOUR_ADMIN_TOKEN"
```

This toggles the `is_admin` status between `true` and `false`.

Response:
```json
{
  "status": "success",
  "message": "Admin status updated successfully",
  "data": {
    "id": "user-456",
    "name": "Jane Doe",
    "is_admin": true
  }
}
```

<Warning>
  Admins cannot toggle their own admin status to prevent accidental lockout.
</Warning>

### Update user password (Admin)

Admins can change any user's password without knowing the old password:

```bash cURL
curl -X PATCH https://api.grancamerino.com/api/users/{user_id}/password \
  -H "Authorization: Bearer YOUR_ADMIN_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "password": "NewPassword123",
    "password_confirmation": "NewPassword123"
  }'
```

### Bulk delete users

```bash cURL
curl -X POST https://api.grancamerino.com/api/users/bulk-delete \
  -H "Authorization: Bearer YOUR_ADMIN_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "user_ids": [
      "user-123e4567-e89b-12d3-a456-426614174001",
      "user-223e4567-e89b-12d3-a456-426614174002",
      "user-323e4567-e89b-12d3-a456-426614174003"
    ]
  }'
```

Response:
```json
{
  "status": "success",
  "message": "3 users deleted successfully",
  "data": {
    "deleted_count": 3,
    "requested_count": 3,
    "skipped_count": 0
  }
}
```

### Get user statistics

```bash cURL
curl https://api.grancamerino.com/api/users/statistics \
  -H "Authorization: Bearer YOUR_ADMIN_TOKEN"
```

Response:
```json
{
  "status": "success",
  "data": {
    "total_users": 1247,
    "total_admins": 5,
    "total_customers": 1242,
    "recent_users": 43,
    "users_with_orders": 856
  }
}
```

## User addresses

Users can manage multiple shipping addresses. Each address can be marked as primary.

### List user addresses

```bash cURL
curl https://api.grancamerino.com/api/addresses \
  -H "Authorization: Bearer YOUR_TOKEN"
```

Response:
```json
{
  "status": "success",
  "data": [
    {
      "id": "addr-123",
      "first_name": "John",
      "last_name": "Doe",
      "phone": "+1234567890",
      "gender": "male",
      "street": "123 Main St",
      "street_line_2": "Apt 4B",
      "city": "New York",
      "state": "NY",
      "postal_code": "10001",
      "country": "United States",
      "is_primary": true,
      "created_at": "2024-03-01T10:00:00.000000Z"
    }
  ]
}
```

### Create an address

```bash cURL
curl -X POST https://api.grancamerino.com/api/addresses \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "first_name": "John",
    "last_name": "Doe",
    "phone": "+1234567890",
    "gender": "male",
    "street": "123 Main St",
    "street_line_2": "Apt 4B",
    "city": "New York",
    "state": "NY",
    "postal_code": "10001",
    "country": "United States",
    "is_primary": true
  }'
```

<Note>
  The `gender` field accepts: `male`, `female`, or `other`. The `street_line_2` field is optional.
</Note>

### Update an address

```bash cURL
curl -X PUT https://api.grancamerino.com/api/addresses/{address_id} \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "street": "456 Oak Avenue",
    "city": "Los Angeles",
    "state": "CA",
    "postal_code": "90001"
  }'
```

All fields are optional when updating.

### Delete an address

```bash cURL
curl -X DELETE https://api.grancamerino.com/api/addresses/{address_id} \
  -H "Authorization: Bearer YOUR_TOKEN"
```

### Set primary address

```bash cURL
curl -X PATCH https://api.grancamerino.com/api/addresses/{address_id}/set-primary \
  -H "Authorization: Bearer YOUR_TOKEN"
```

This automatically sets all other addresses to non-primary.

### Get user addresses (Admin)

Admins can view any user's addresses:

```bash cURL
curl https://api.grancamerino.com/api/users/{user_id}/addresses \
  -H "Authorization: Bearer YOUR_ADMIN_TOKEN"
```

## User favorites

Users can save products to their favorites list.

### List favorites

```bash cURL
curl https://api.grancamerino.com/api/favorites \
  -H "Authorization: Bearer YOUR_TOKEN"
```

Response:
```json
{
  "status": "success",
  "data": [
    {
      "id": "fav-123",
      "product_id": "prod-456",
      "product": {
        "id": "prod-456",
        "name": "Real Madrid Home Jersey 2024",
        "slug": "real-madrid-home-2024",
        "price_usd": 89.99,
        "price_cop": 350000,
        "images": [...],
        "variants": [...]
      },
      "created_at": "2024-03-01T10:00:00.000000Z"
    }
  ]
}
```

### Add to favorites

```bash cURL
curl -X POST https://api.grancamerino.com/api/favorites \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "product_id": "prod-123e4567-e89b-12d3-a456-426614174000"
  }'
```

If the product is already in favorites:
```json
{
  "status": "error",
  "message": "Product already in favorites"
}
```

### Remove from favorites

```bash cURL
curl -X DELETE https://api.grancamerino.com/api/favorites/{product_id} \
  -H "Authorization: Bearer YOUR_TOKEN"
```

### Toggle favorite status

Add or remove a product from favorites in a single request:

```bash cURL
curl -X POST https://api.grancamerino.com/api/favorites/{product_id}/toggle \
  -H "Authorization: Bearer YOUR_TOKEN"
```

Response:
```json
{
  "status": "success",
  "message": "Product added to favorites",
  "data": {
    "is_favorited": true,
    "favorite": {
      "id": "fav-123",
      "product": {...}
    }
  }
}
```

If the product was already favorited, it will be removed and `is_favorited` will be `false`.

### Get user favorites (Admin)

Admins can view any user's favorites:

```bash cURL
curl https://api.grancamerino.com/api/users/{user_id}/favorites \
  -H "Authorization: Bearer YOUR_ADMIN_TOKEN"
```

Returns paginated results (20 items per page).

## User orders

Admins can view all orders for a specific user:

```bash cURL
curl https://api.grancamerino.com/api/users/{user_id}/orders \
  -H "Authorization: Bearer YOUR_ADMIN_TOKEN"
```

Response includes full order details with items, products, and variant information. Results are paginated with 20 orders per page.

## User roles and permissions

The API has two user roles:

### Client role
- Register and authenticate
- Manage own profile
- Create and manage own addresses
- Create and manage favorites
- Create and view own orders
- Cancel own orders (if not shipped/delivered)

### Admin role
- All client permissions
- Manage all users (create, update, delete)
- Manage products, categories, teams, etc.
- View all orders from all users
- Update order statuses
- Bulk operations (price updates, user deletion)
- Access to statistics and analytics

<Note>
  Admin-only endpoints are protected by the `admin` middleware in addition to authentication. Non-admin users receive a 403 Forbidden response.
</Note>
