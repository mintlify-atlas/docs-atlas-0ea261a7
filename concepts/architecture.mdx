The Gran Camerino E-commerce API is built on Laravel and follows a clean MVC architecture with dedicated service layers for external integrations.

## Laravel MVC structure

The application follows Laravel's Model-View-Controller pattern, adapted for a REST API:

- **Models**: Eloquent ORM models with relationships and business logic
- **Controllers**: HTTP request handlers in `app/Http/Controllers/`
- **Services**: External API integrations in `app/Services/`
- **Middleware**: Authentication and authorization in `app/Http/Middleware/`
- **Routes**: API endpoint definitions in `routes/api.php`

### Controller organization

Controllers are organized by resource type and follow RESTful conventions:

- `AuthController.php` - User authentication (login, register, logout)
- `ProductController.php` - Product catalog management
- `OrderController.php` - Order processing and payment integration
- `UserController.php` - User management (admin only)
- `CategoryController.php`, `TeamController.php`, etc. - Resource management

All controllers extend the base `Controller` class and return JSON responses with consistent structure:

```php
return response()->json([
    'status' => 'success',
    'data' => $data
]);
```

## Service layer

External integrations are encapsulated in service classes to maintain clean separation of concerns.

### WompiService

Handles Colombian payment gateway integration (`app/Services/WompiService.php:8`):

```php
class WompiService
{
    private string $publicKey;
    private string $privateKey;
    private string $integritySecret;
    private string $baseUrl;

    public function __construct()
    {
        $this->publicKey = config('services.wompi.public_key');
        $this->privateKey = config('services.wompi.private_key');
        $this->integritySecret = config('services.wompi.integrity_secret');
        $this->baseUrl = config('services.wompi.url');
    }
}
```

**Key methods:**
- `generateIntegritySignature()` - Creates SHA256 signatures for transactions (app/Services/WompiService.php:26)
- `createPaymentData()` - Prepares checkout widget data (app/Services/WompiService.php:75)
- `buildCheckoutUrl()` - Constructs Wompi payment URLs (app/Services/WompiService.php:131)
- `verifyEventChecksum()` - Validates webhook signatures (app/Services/WompiService.php:239)

### CloudinaryService

## Request flow

A typical API request flows through these layers:

1. **Route** (`routes/api.php`) - Matches URL pattern and applies middleware
2. **Middleware** - Validates JWT token, checks user role
3. **Controller** - Validates request data, orchestrates business logic
4. **Service** (if needed) - Handles external API calls
5. **Model** - Database operations via Eloquent ORM
6. **Response** - Returns JSON with consistent structure

Example order creation flow (`app/Http/Controllers/OrderController.php:105`):

```php
public function store(Request $request): JsonResponse
{
    $validated = $request->validate([...]);

    return DB::transaction(function () use ($validated, $request) {
        // 1. Calculate order totals
        // 2. Apply coupon if provided
        // 3. Create order record
        // 4. Create order items
        // 5. Generate payment link (Wompi or NOWPayments)
        // 6. Return order with checkout URL
    });
}
```
