## Authentication

Requires authentication via Bearer token. Users can only cancel their own orders.

## Path Parameters

<ParamField path="order" type="string" required>
  Order UUID
</ParamField>

## Cancellation Rules

Orders can only be cancelled if they are NOT in the following statuses:
- `shipped` - Order has already been shipped
- `delivered` - Order has been delivered

Orders in the following statuses CAN be cancelled:
- `pending` - Awaiting payment
- `paid` - Payment confirmed but not yet shipped
- `preparing` - Being prepared for shipment
- `in_transit` - In transit (use with caution)

<Note>
Cancelling a `paid` order requires coordination with the payment gateway for refunds. The API will update the order status to `cancelled`, but refund processing must be handled separately.
</Note>

## Cancellation Workflow

### For Pending Orders (Not Paid)

1. User requests cancellation via API
2. Order status is updated to `cancelled`
3. No refund processing needed
4. Inventory is restored (if inventory tracking is enabled)

### For Paid Orders (Before Shipment)

1. User requests cancellation via API
2. Order status is updated to `cancelled`
3. **Admin must process refund manually** via payment gateway:
   - **Wompi**: Process refund through Wompi dashboard or API
   - **NOWPayments**: Contact NOWPayments support for crypto refunds
4. Inventory is restored (if inventory tracking is enabled)
5. User is notified of cancellation and refund timeline

### Orders That Cannot Be Cancelled

Orders with status `shipped` or `delivered` cannot be cancelled through this endpoint. These require:
- Return/refund request through customer service
- RMA (Return Merchandise Authorization) process
- Admin intervention

<Warning>
This endpoint only updates the order status to `cancelled`. It does **NOT** automatically process refunds. Refunds must be handled separately based on the payment gateway used.
</Warning>
