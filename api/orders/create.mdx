## Authentication

Requires authentication via Bearer token.

## Request Body

<ParamField body="address_id" type="string" required>
  UUID of the shipping address
</ParamField>

<ParamField body="coupon_code" type="string">
  Coupon code to apply discount (e.g., `WELCOME10`)
</ParamField>

<ParamField body="items" type="array" required>
  Array of order items (minimum 1 item)

  <Expandable title="Item Object">
    <ParamField body="product_id" type="string" required>
      Product UUID
    </ParamField>

    <ParamField body="product_variant_id" type="string" required>
      Product variant UUID (size)
    </ParamField>

    <ParamField body="quantity" type="integer" required>
      Quantity (minimum 1)
    </ParamField>

    <ParamField body="customization_name" type="string">
      Player name for jersey customization (max 255 characters)
    </ParamField>

    <ParamField body="customization_number" type="string">
      Jersey number for customization (max 50 characters)
    </ParamField>
  </Expandable>
</ParamField>

<ParamField body="payment_gateway" type="string" required>
  Payment gateway: `wompi` (COP), `nowpayments` (USD crypto), or `paymentnow`
</ParamField>

<ParamField body="crypto_currency" type="string">
  Cryptocurrency for NOWPayments (e.g., `btc`, `eth`, `usdt`). Required when using `nowpayments`.
</ParamField>

<ParamField body="redirect_url" type="string">
  Custom redirect URL after payment. If not provided, uses default configuration. Note: Wompi sandbox only accepts `http://localhost` (not `https://localhost`).
</ParamField>

<ParamField body="notes" type="string">
  Additional notes for the order
</ParamField>

## Response

<ResponseField name="status" type="string">
  Response status (`success`)
</ResponseField>

<ResponseField name="message" type="string">
  Success message
</ResponseField>

<ResponseField name="data" type="object">
  Order and payment information

  <Expandable title="Data Object">
    <ResponseField name="order" type="object">
      Order object with items, address, and coupon
    </ResponseField>

    <ResponseField name="payment_gateway" type="string">
      Payment gateway used
    </ResponseField>

    <ResponseField name="payment_data" type="object">
      Payment gateway-specific data
      
      **Wompi:**
      - `reference`: Order reference
      - `amount_in_cents`: Amount in cents (COP)
      - `currency`: `COP`
      - `signature`: Integrity signature
      - `public_key`: Wompi public key
      - `redirect_url`: Return URL after payment

      **NOWPayments:**
      - `invoice_id`: Invoice ID
      - `invoice_url`: Payment page URL
      - `order_id`: Order reference
      - `price_amount`: Amount in USD
      - `price_currency`: `usd`
      - `pay_currency`: Cryptocurrency selected
    </ResponseField>

    <ResponseField name="payment_urls" type="object">
      Payment URLs for the selected gateway
      
      **Wompi:**
      - `checkout_url`: Direct checkout URL with pre-filled data
      - `widget_script`: Wompi widget script URL

      **NOWPayments:**
      - `payment_url`: Invoice payment page
      - `invoice_url`: Same as payment_url
    </ResponseField>

    <ResponseField name="order_id" type="string">
      Order UUID
    </ResponseField>

    <ResponseField name="order_number" type="string">
      Human-readable order number
    </ResponseField>

    <ResponseField name="total_amount" type="number">
      Final total amount
    </ResponseField>

    <ResponseField name="checkout_url" type="string">
      Direct checkout URL (same as `payment_urls.checkout_url` or `payment_urls.payment_url`)
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="debug" type="object">
  Debug information for Wompi signature verification (remove in production)
</ResponseField>

## Payment Flow

### Wompi (Colombian Pesos - COP)

1. Order is created with status `pending`
2. Payment data is generated with integrity signature
3. Client redirects user to `checkout_url` or uses Wompi widget
4. User completes payment on Wompi
5. Wompi webhook updates order status to `paid` or `failed`

### NOWPayments (Cryptocurrency - USD)

1. Order is created with status `pending`
2. Invoice is created via NOWPayments API
3. Client redirects user to `invoice_url`
4. User selects cryptocurrency and completes payment
5. NOWPayments webhook updates order status based on payment status:
   - `finished` → `paid`
   - `failed`, `expired`, `refunded` → `failed`
   - `waiting`, `confirming`, `confirmed`, `sending` → `pending`

## Order Status Values

- `pending`: Order created, awaiting payment
- `paid`: Payment confirmed
- `preparing`: Order being prepared for shipment
- `shipped`: Order shipped
- `in_transit`: Order in transit
- `delivered`: Order delivered to customer
- `cancelled`: Order cancelled
- `failed`: Payment failed
