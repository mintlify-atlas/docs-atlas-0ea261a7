## Authentication

Requires authentication via Bearer token with **admin role**.

## Path Parameters

<ParamField path="order" type="string" required>
  Order UUID
</ParamField>

## Request Body

<ParamField body="status" type="string" required>
  New order status. Must be one of:
  - `pending` - Order created, awaiting payment
  - `paid` - Payment confirmed
  - `preparing` - Order being prepared for shipment
  - `shipped` - Order has been shipped
  - `in_transit` - Order is in transit
  - `delivered` - Order delivered to customer
  - `cancelled` - Order cancelled
  - `failed` - Payment failed
</ParamField>

## Automatic Timestamp Updates

When updating status, the following timestamps are automatically set:

- Setting status to `paid`: Updates `paid_at` if not already set
- Setting status to `shipped`: Updates `shipped_at` if not already set
- Setting status to `delivered`: Updates `delivered_at` if not already set

## Status Flow

Typical order status progression:

1. `pending` - Order created, awaiting payment
2. `paid` - Payment confirmed by webhook
3. `preparing` - Admin marks order as being prepared
4. `shipped` - Admin marks order as shipped
5. `in_transit` - Order in transit (optional)
6. `delivered` - Order delivered to customer

Alternative flows:
- `pending` → `failed` - Payment failed
- `pending` → `cancelled` - Order cancelled before payment
- `paid` → `cancelled` - Order cancelled after payment (requires refund)
