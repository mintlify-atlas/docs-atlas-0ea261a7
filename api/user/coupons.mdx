## Overview

The Coupons API provides functionality for managing promotional discount coupons. Users can list active coupons and validate coupon codes before checkout. Administrators can create, update, and delete coupons.

Coupons support two discount types:
- **Percentage**: Discount based on percentage of order total (e.g., 20% off)
- **Fixed**: Fixed amount discount (e.g., $10 off)

**Authentication:** 
- List, view, and validate endpoints: Available to all authenticated users
- Create, update, delete endpoints: Admin access required

## Validate Coupon

<api method="POST" path="/api/coupons/validate" auth="required">
Validate a coupon code and calculate the discount for a given order amount. This endpoint checks all validation rules and returns the discount details.
</api>

### Request Body

<ParamField name="code" type="string" required>
The coupon code to validate (case-insensitive)
</ParamField>

<ParamField name="amount" type="number" required>
The order total amount to apply the coupon to (must be >= 0)
</ParamField>

### Response

<ResponseField name="status" type="string">
Response status ("success" or "error")
</ResponseField>

<ResponseField name="message" type="string">
Validation result message
</ResponseField>

<ResponseField name="data" type="object">
Discount calculation details (only on success)
  <Expandable title="Validation data">
    <ResponseField name="coupon" type="object">The coupon object</ResponseField>
    <ResponseField name="discount_amount" type="number">Calculated discount amount</ResponseField>
    <ResponseField name="final_amount" type="number">Order total after discount (minimum 0)</ResponseField>
  </Expandable>
</ResponseField>

### Validation Rules

A coupon is considered valid only if ALL of the following conditions are met:

1. Coupon code exists in the database
2. `is_active` is true
3. `starts_at` is null or in the past
4. `expires_at` is null or in the future
5. `usage_limit` is null or `used_count < usage_limit`
6. Order amount meets `minimum_amount` requirement (if set)

### Discount Calculation

- **Percentage coupons**: `discount = amount * (value / 100)`
- **Fixed coupons**: `discount = min(value, amount)` (never exceeds order total)
- If order amount is below `minimum_amount`, discount is 0

### Example Request (Valid Coupon)

```bash
curl -X POST https://api.grancamerino.com/api/coupons/validate \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -H "Content-Type: application/json" \
  -H "Accept: application/json" \
  -d '{
    "code": "SUMMER2024",
    "amount": 100.00
  }'
```

## Create Coupon

<api method="POST" path="/api/coupons" auth="admin">
Create a new coupon. Requires admin authentication.
</api>

### Request Body

<ParamField name="code" type="string" required>
Unique coupon code (will be converted to uppercase). Must be unique across all coupons.
</ParamField>

<ParamField name="name" type="string" required>
Display name of the coupon (max 255 characters)
</ParamField>

<ParamField name="description" type="string">
Optional description of the coupon
</ParamField>

<ParamField name="type" type="string" required>
Discount type. Must be one of: `percentage`, `fixed`
</ParamField>

<ParamField name="value" type="number" required>
Discount value (must be >= 0)
- For percentage: value between 0-100
- For fixed: amount in dollars
</ParamField>

<ParamField name="minimum_amount" type="number">
Minimum order amount required to use coupon (must be >= 0)
</ParamField>

<ParamField name="usage_limit" type="integer">
Maximum number of times the coupon can be used (must be >= 1)
</ParamField>

<ParamField name="starts_at" type="string">
Start date and time (ISO 8601 format)
</ParamField>

<ParamField name="expires_at" type="string">
Expiration date and time (ISO 8601 format). Must be after `starts_at` if both are provided.
</ParamField>

<ParamField name="is_active" type="boolean" default="true">
Whether the coupon is active
</ParamField>

### Response

<ResponseField name="status" type="string">
Response status ("success")
</ResponseField>

<ResponseField name="message" type="string">
Success message
</ResponseField>

<ResponseField name="data" type="object">

</ResponseField>

## Coupon Types Explained

### Percentage Coupons

Percentage coupons apply a percentage discount to the order total:

- **Value**: Represents the percentage (0-100)
- **Example**: value = 20 means 20% off
- **Calculation**: `discount = amount * (value / 100)`
- **Use case**: General sales, category-specific promotions

```json
{
  "code": "SUMMER20",
  "type": "percentage",
  "value": 20.00
}
// Order $100 → Discount $20 → Final $80
```

### Fixed Amount Coupons

Fixed coupons apply a specific dollar amount discount:

- **Value**: Represents the fixed discount amount
- **Example**: value = 10 means $10 off
- **Calculation**: `discount = min(value, amount)`
- **Use case**: Free shipping, welcome bonuses, referral credits

```json
{
  "code": "SAVE15",
  "type": "fixed",
  "value": 15.00
}
// Order $100 → Discount $15 → Final $85
// Order $10 → Discount $10 → Final $0 (capped at order total)
```
