## Get up and running

This guide will walk you through making your first API calls to Gran Camerino. You'll register a user, authenticate, and retrieve products.

<Note>
  This quick start assumes you have the API already installed and running. If not, see the [installation guide](/installation) first.
</Note>

## Prerequisites

Before you begin, make sure you have:

- The Gran Camerino API running (default: `http://localhost:8000`)
- A tool to make HTTP requests (curl, Postman, or similar)
- The API base URL (we'll use `http://localhost:8000/api` in examples)

## Make your first request

<Steps>
  <Step title="Register a new user">
    Create a new user account to get started. The API will return a JWT token for authentication.

    ```bash
    curl -X POST http://localhost:8000/api/register \
      -H "Content-Type: application/json" \
      -d '{
        "name": "John Doe",
        "email": "john@example.com",
        "password": "password123"
      }'
    ```

    **Response:**
    ```json
    {
      "message": "User registered successfully",
      "user": {
        "id": "9d3e8b1a-4f2c-4d5e-8a9b-1c2d3e4f5a6b",
        "name": "John Doe",
        "email": "john@example.com",
        "role": "client"
      },
      "token": "eyJ0eXAiOiJKV1QiLCJhbGciOiJIUzI1NiJ9...",
      "token_type": "bearer",
      "expires_in": 3600
    }
    ```

    <Tip>
      Save the `token` value - you'll need it for authenticated requests.
    </Tip>
  </Step>

  <Step title="Browse products">
    Retrieve the product catalog. This endpoint is public and doesn't require authentication.

    ```bash
    curl -X GET "http://localhost:8000/api/products?per_page=10" \
      -H "Accept: application/json"
    ```

    **Response:**
    ```json
    {
      "status": "success",
      "data": {
        "current_page": 1,
        "data": [
          {
            "id": "9d3e8b1a-4f2c-4d5e-8a9b-1c2d3e4f5a6b",
            "name": "Real Madrid Home Jersey 2024",
            "slug": "real-madrid-home-2024",
            "price_usd": 89.99,
            "price_cop": 350000,
            "description": "Official Real Madrid home jersey",
            "is_active": true,
            "category": {
              "id": "...",
              "name": "Jerseys"
            },
            "team": {
              "id": "...",
              "name": "Real Madrid",
              "country": {
                "name": "Spain"
              }
            },
            "images": [
              {
                "id": "...",
                "url": "https://res.cloudinary.com/.../image.jpg",
                "is_primary": true
              }
            ],
            "variants": [
              {
                "id": "...",
                "size": {
                  "name": "M"
                }
              }
            ]
          }
        ],
        "per_page": 10,
        "total": 150
      }
    }
    ```

    <Tip>
      You can filter products by category, team, country, league, price range, and more. See the [Products API reference](/api/products/list) for all available filters.
    </Tip>
  </Step>

  <Step title="Get product details">
    Retrieve detailed information about a specific product.

    ```bash
    curl -X GET http://localhost:8000/api/products/{product_id} \
      -H "Accept: application/json"
    ```

    Replace `{product_id}` with the actual product ID from the previous response.
  </Step>

  <Step title="Add a product to favorites">
    Now let's use your authentication token to add a product to your favorites.

    ```bash
    curl -X POST http://localhost:8000/api/favorites \
      -H "Content-Type: application/json" \
      -H "Authorization: Bearer YOUR_TOKEN_HERE" \
      -d '{
        "product_id": "9d3e8b1a-4f2c-4d5e-8a9b-1c2d3e4f5a6b"
      }'
    ```

    **Response:**
    ```json
    {
      "status": "success",
      "message": "Product added to favorites",
      "data": {
        "id": "...",
        "product_id": "9d3e8b1a-4f2c-4d5e-8a9b-1c2d3e4f5a6b",
        "user_id": "..."
      }
    }
    ```

    <Warning>
      Don't forget to replace `YOUR_TOKEN_HERE` with the actual token from Step 1.
    </Warning>
  </Step>

  <Step title="Create an order">
    Place an order with the products you want to purchase.

    ```bash
    curl -X POST http://localhost:8000/api/orders \
      -H "Content-Type: application/json" \
      -H "Authorization: Bearer YOUR_TOKEN_HERE" \
      -d '{
        "items": [
          {
            "product_variant_id": "variant-uuid-here",
            "quantity": 1,
            "price_usd": 89.99,
            "price_cop": 350000
          }
        ],
        "address_id": "address-uuid-here",
        "payment_method": "wompi",
        "currency": "cop",
        "total_usd": 89.99,
        "total_cop": 350000
      }'
    ```

    The API will create the order and return payment information including a checkout URL.
  </Step>
</Steps>

## Authentication flow

Understand how authentication works in the API:

<Steps>
  <Step title="Register or login">
    Use `/api/register` to create a new account or `/api/login` to authenticate an existing user.
  </Step>
  
  <Step title="Include token in requests">
    Add the JWT token to the `Authorization` header as `Bearer YOUR_TOKEN`.
  </Step>
  
  <Step title="Refresh when needed">
    Use `/api/refresh` to get a new token before the current one expires (default: 60 minutes).
  </Step>
  
  <Step title="Logout">
    Call `/api/logout` to invalidate the token when done.
  </Step>
</Steps>
